
## Testing a new Top-Level Key

Once you have your development branch with the desired changes (see [README.md](./../tap-pkg/README.md)), you can validate 
whether the expected outcome is achieved by running `ytt` and also, by installing a custom TAP packages bundle in 
a kubernetes cluster.

Unit tests were added to cover the top level key `shared.ingress_domain` scenarios. See section [Unit Tests](#unit-tests).

### Unit tests

Run the makefile command below to run the existing unit tests.

```shell
make unit-tests-yaml
```

### Quick feedback with YTT

1. Create a single or multiple tap-values.yaml config files and place them in a folder 
   i.e. tap-values-test/tap-values-light.yaml
2. Set the enviroment variable `TAP_VALUES_TEST` with the location of the files created in the above step
```shell
export TAP_VALUES_TEST=tap-values-test
```

4. Run the make target `test-tap-values`
```shell
make test-tap-values
```

5. Under the folder `out` (created by the make target in the repository root folder), you will find the final yamls containing the changes you
   made if no templating errors were raised. After running the commands above, make sure to:
   - Check for any templating errors
   - Test with various permutations of shared and package specific config. i.e. experiment with different content 
   for tap-values.yaml

### Custom TAP installation

#### Pre-requisites

For the instructions below, we assume you have a working TAP installation in a cluster.

Refer to [TAP docs](https://docs.vmware.com/en/VMware-Tanzu-Application-Platform/index.html) to install TAP 
in a kubernetes cluster.

#### Testing your changes using an existing TAP installation

1. Once you have a branch with the desired changes, create an imgpkg bundle with ./tap-pkg contents and
   take note of the bundle location from the command output below.

```shell
export MY_REGISTRY_REPO="projects.registry.vmware.com/myproject/test"
export TAP_VERSION="1.1.0-build.12"
export MY_CUSTOM_TAP_VERSION="${TAP_VERSION}-yourname"

imgpkg push -b ${MY_REGISTRY_REPO}:${MY_CUSTOM_TAP_VERSION} -f tap-pkg
```

2. Copy the PackageCR from the latest TAP build and rename it.
```shell
cp packages/tap/${TAP_VERSION}.yaml packages/tap/${MY_CUSTOM_TAP_VERSION}.yaml
```

3. Edit your recently created package yaml:
   1. Update package.yaml#.metadata.name
   2. Update package.yaml#.spec.version
   3. Replace the imgpkg bundle in your recently created package.yaml with the one generated by 
   the previous step: package.yaml#.spec.template.fetch[0].imgpkgBundle.image

```yaml
apiVersion: data.packaging.carvel.dev/v1alpha1
kind: Package
metadata:
  name: tap.tanzu.vmware.com.${MY_CUSTOM_TAP_VERSION}
spec:
  refName: tap.tanzu.vmware.com
  version: ${MY_CUSTOM_TAP_VERSION}
  template:
    spec:
      fetch:
        - imgpkgBundle: ${IMGPKG_PUSH_OUTPUT} 
```

4. Apply your custom TAP package to an existing TAP cluster.

```shell
kubectl apply -f packages/tap/${MY_CUSTOM_TAP_VERSION}.yaml -n tap-install

# confirm it contains your custom imgpkg bundle
kubectl get pkg tap.tanzu.vmware.com.${MY_CUSTOM_TAP_VERSION} -n tap-install -ojsonpath='{.spec.template.spec.fetch[0].imgpkgBundle.image}'
```

5. Edit the PackageInstall resource to grab your custom version.

```shell
kubectl edit -n tap-install pkgi tap
```

```yaml
apiVersion: packaging.carvel.dev/v1alpha1
kind: PackageInstall
...
spec:
  packageRef:
    refName: tap.tanzu.vmware.com
    versionSelection:
      constraints: ${MY_CUSTOM_TAP_VERSION}
```


*Note*: Optional: You can use the command below to force an update

```shell
kubectl patch -n tap-install pkgi tap --type merge --patch '{"spec": {"paused": true}}' && \
kubectl patch -n tap-install pkgi tap --type merge --patch '{"spec": {"paused": false}}'
```

6. Confirm the packages were successfully reconciled

```shell
# watches for changes in all PackageInstall resources
kubectl get pkgi -n tap-install -w
```

7. You can test different permutations of tap-values.yaml file and see if your changes work as expected.

```shell
tanzu package installed update tap --values-file tap-values.yaml -n tap-install
```

8. Confirm if the correct values were passed to individual packages / components.

```shell
# tap values
kubectl get secret tap-tap-install-values -n tap-install -ojsonpath="{.data.tap-values-light\.yaml}" | base64 -d

# examples of individual components values
kubectl get secret accelerator-values -n tap-install -ojsonpath="{.data.values\.yaml}" | base64 -d
kubectl get secret appliveview-connector-values -n tap-install -ojsonpath="{.data.values\.yml}" | base64 -d 
kubectl get secret appliveview-values -n tap-install -ojsonpath="{.data.values\.yml}" | base64 -d
kubectl get secret cnrs-values -n tap-install -ojsonpath="{.data.values\.yaml}" | base64 -d
kubectl get secret tap-gui-values -n tap-install -ojsonpath="{.data.values\.yaml}" | base64 -d
```
